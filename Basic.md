##1. Basic 
###1.1. Как начинается php код, с помощью каких символов

Вариантов оформления php кода несколько:

 - Использование ``<?php`` в начале скрипта. Php код чаще всего начинаться со строк ``<?php``, рекомендуется использовать именно этот синтаксис. Например, скрипт может выглядеть так:
```
<?php
	//code
?>
```
 - Использование ``<?`` в начале скрипта. Данный синтаксис возможен, когда в файле php.ini заданы настройки ``short_open_tag = 1``. Рекомендуется после изменения файла php.ini перезапустить веб-сервер и php-fpm, если он используется.  Например, можно писать вот так:
```
<?
	//code
?>
```
Выдержка из документации php.net
> **short_open_tag** boolean
Определяет, разрешается ли короткая форма записи (``<? ?>``) тегов PHP. Если вы хотите использовать PHP >совместно с XML, вы можете отключить эту опцию, чтобы беспрепятственно использовать ``<?xml ?>``. В >противном случае, вы можете отобразить это при помощи PHP, например: ``<?php echo '<?xml version="1.0"?>'; ?>``. Если же эта опция отключена, вы должны использовать длинную форму открывающего тега PHP (``<?php ?>``).

Короткие теги в php удобно использовать в конструкциях ``<?= $value ?>``, что заменяет следующий код: ``<?php echo $vaule ?>``.

- Использование ASP тегов позволяет писать php код в виде ``<%``. Данный синтаксис также доступен только при включении директивы ``asp_tags = 1`` в файле php.ini. Например, код можно писать вот так:
```
<%
	//code
%>
```
**Важно!** Стоит отметить, что начиная с версии PHP 7.0.0 директива ``asp_tags`` удалена, данная возможность написания синтаксиса отсутствует.

Выдержка из документации php.net:
> **asp_tags** boolean
Включает использование тегов в стиле ASP в дополнение к обычным тегам ``<?php ?>``. Это касается и сокращения для отображения значения переменных ``<%= $value %>``. Для получения дополнительной информации, обратитесь к разделу Экранирование HTML.

- Использование тегов ``<script language="php">`` также позволяет писать php код прямо на странице. Например, скрипт может выглядеть так:
```
<script language="php">
	// code
</script>
```

Также важным моментом является закрывающий тег ``?>``, который можно не всегда использовать в ваших скриптах. Чаще всего, при использовании MVC, его используют только в файлах представлений, а в файлах моделей и контроллеров не используют.
**Правило:** если в файле скрипта .php присутствуют html теги и вставки php кода, то использование закрывающего тега ``?>`` обязательно. Иначе рекомендуется отказаться от использования закрывающего тега.

Рекомендуется дополнительно изучить статью "Изолирование от HTML" (http://php.net/manual/ru/language.basic-syntax.phpmode.php)

### 1.2. Какие оформляются комментарии в коде

Источник: http://php.net/manual/ru/language.basic-syntax.comments.php

Синтаксис PHP поддерживает следующие символы для обозначения комментариев:

- Использование символов ``//``, которые комментируют одну строку, код может выглядеть так:
```
<?php
	// code comment
	echo 'test';
?>
```
- Использование символа ``#``,  который комментирует одну строку, код может выглядеть так:
```
<?php
	# code comment
	echo 'test';
?>
```
- Использование символов ``/* comments */``, которые позволяют комментировать несколько строк сразу, например:
```
<?php
	/*
		comment 1
		comment 2
	*/
?>
```
Чаще всего в таких комментариях используют PHPDoc для документирования и описания методов/классов, например:

### 1.3. Как объявляется константа

Источник: http://php.net/manual/ru/language.constants.php

Константы - это идентификаторы (имена) простых значений. Исходя из их названия, нетрудно понять, что их значение не может изменяться в ходе выполнения скрипта (исключения представляют "волшебные" константы, которые на самом деле не являются константами в полном смысле этого слова). Имена констант чувствительны к регистру. По принятому соглашению, имена констант всегда пишутся в верхнем регистре.

Имя константы должно соответствовать тем же правилам, что и другие имена в PHP. Правильное имя начинается с буквы или символа подчеркивания и состоит из букв, цифр и подчеркиваний. Регулярное выражение для проверки правильности имени константы выглядит так: ``[a-zA-Z_\x7f-\xff][a-zA-Z0-9_\x7f-\xff]*``

Принято правило, что константы необходимо именовать только с использованием заглавных букв, разделяю слова символом подчеркивания, например ``TEST_CONST``.

Константы в коде можно объявлять с использованием следующего синтаксиса:

- Использование функции ``define()``, которая создает переменную с нужным вам названием и значением. Код может выглядеть так:
```
<?php
	define("FOO",     "something");
	define("FOO2",    "something else");
	define("FOO_BAR", "something more");
	
	echo FOO_BAR;
?>
```
- Использование ключевого слова ``const`` в объявлении класса. Например, можно использовать следующий код:
```
<?php
class Constants
{
  //define('MIN_VALUE', '0.0');  WRONG - Works OUTSIDE of a class definition.
  //define('MAX_VALUE', '1.0');  WRONG - Works OUTSIDE of a class definition.

  const MIN_VALUE = 0.0;      // RIGHT - Works INSIDE of a class definition.
  const MAX_VALUE = 1.0;      // RIGHT - Works INSIDE of a class definition.

  public static function getMinValue()
  {
    return self::MIN_VALUE;
  }

  public static function getMaxValue()
  {
    return self::MAX_VALUE;
  }
}

#Example 1:
$min = Constants::MIN_VALUE;
$max = Constants::MAX_VALUE;

#Example 2:
$min = Constants::getMinValue();
$max = Constants::getMaxValue();
?>
```

Не стоит именовать константы вида ``__CONST_NAME__``, так как сам PHP объявляет множество волшебных констант, может возникнуть конфликт. Волшебные константы доступны в каждом скрипте без их инициализации, более подробно про них можно прочитать тут: http://php.net/manual/ru/language.constants.predefined.php .

Стоит отметить, что константы можно переопределять в дочернем классе, например:
```
<?php
	class Foo {
		const FOO_CONST = 'foo';
		const ONLY_FOO_CONST = 'only foo';
	}
	class Bar extends Foo {
		const FOO_CONST = 'bar';
	}

	echo Bar::FOO_CONST; // выведет bar
	echo Bar::ONLY_FOO_CONST; // выведет only foo
?>
```
